# Pokémon Data (寶可夢相關)

## Where the data lives — two independent copies, always edit both

| File | Role |
|---|---|
| `pokemon_battle.html` | Single-player source of truth. `public/single.html` is a byte-for-byte mirror — after editing `pokemon_battle.html`, always run `cp pokemon_battle.html public/single.html` and verify with `diff`. |
| `server.js` | PvP source of truth (server-authoritative). Has its own separate `POKEMON`/`TRAINERS` arrays — **not shared** with the HTML files. `public/pvp.html` has no Pokémon data of its own; it only renders whatever the server broadcasts. |

There is no shared data module — this is 4 static HTML/JS files, no build step. Any change to a Pokémon's stats/moves/ability must be made **twice**: once in `pokemon_battle.html` (then synced to `single.html`), once in `server.js`.

## POKEMON entry shape

```js
{ id:26, name:'雷丘', type:'electric', type2:null /* optional */, hp:200, tier:1 /* single-player only, 1-3 */,
  ability:{id:'static', name:'靜電', trigger:'onDefend', desc:'受到攻擊後 20% 機率讓攻擊者陷入麻痺'}, // optional, see Abilities below
  mega:{spriteId:10033, type:'grass', type2:'poison', ability:{id:'thick-fat', name:'厚脂肪', trigger:'onDefend', desc:'...'}}, // optional, see Mega Evolution below
  attacks:[
    {name:'十萬伏特', dmg:40, cost:1, type:'electric', status:{effect:'paralysis', chance:0.30}}, // status optional
    ...always exactly 4 attacks: 2 "weak" (low cost, cost floor ≤ per-turn energy regen) + 2 "strong" (high cost, requires banking energy)
  ]
}
```
`cost` (0-20, added 2026-07-02) is the energy the move consumes — see battle-logic skill's Energy System section for the resource itself. Every Pokémon's 4 moves are deliberately split weak/strong by cost, not flat — don't add a 5th move or break the 2-weak/2-strong shape without deliberately redesigning the tier cost table below.
- `id` is the PokéAPI id — sprites are fetched live from `https://raw.githubusercontent.com/PokeAPI/sprites/...{id}.png` (animated GIF preferred, static PNG fallback), no local assets, no API key. Must be unique across the whole `POKEMON` array (it's used as a set key in roster/swap-candidate generation) — check for collisions against the *real* Pokédex number before adding a species, not just against names (two sessions have picked a species whose real dex number already existed in the roster under a different Chinese name).
- Single-player only: `tier` (1=弱/2=中/3=強, used by `launchStage()` to build the CPU's deck for that stage).
- Roster generation (single-player `pickRandomRoster()`, PvP `randomRoster()` in server.js) — both cap Pokémon with **HP ≥ 300 to at most 1 per generated roster** (added 2026-07-03, shared logic in both engines: shuffle, then skip any HP≥300 candidate once the cap is already filled). Applies to single-player's initial 5-roster draw, its swap-candidate draws, and PvP's initial 6-roster + reroll.
- Battle-instance cloning: single-player `clone()`, PvP `clonePoke()` — both do a shallow `{...p}` plus deep-copy `attacks` and add `cur` (current HP) / `status`. A shallow spread means `ability`/`mega` carry over automatically — don't need to special-case them when cloning. Both `clone()`/`clonePoke()` additionally initialize `megaEvolved:false` **only if** `p.mega` exists (`undefined` otherwise). **Mega energy itself is not a per-instance field** — it's a per-side pool on `G` (`G.playerMegaEnergy`/`G.cpuMegaEnergy`, shared across the whole team like move-energy), reworked from an earlier per-Pokémon design the same day it shipped; see battle-logic skill's Mega Evolution section for the full mechanic, including the one-mega-per-battle rule and the full-heal-on-evolve behavior.
- **`clone()` also sets `baseId: p.id` on every Pokémon** (single-player only — added 2026-07-05, PvP has no equivalent need, see below), unconditionally, not just for Mega-capable species. Once a Pokémon's identity is mutated in place (Mega Evolution overwrites `.id` with the Mega sprite id — not a real dex number, so it won't be found in `POKEMON`), any code that needs to look up the *original* species entry again must key off `baseId`, never `id`. Two real bugs came from forgetting this: `pickEnemy()` (picking a defeated CPU Pokémon to add to your roster) used to do `POKEMON.find(p => p.id === picked.id)`, which silently found nothing and dropped the Pokémon entirely if it had Mega-evolved during that battle (fixed 2026-07-05 to use `picked.baseId ?? picked.id`). `launchStage()`'s per-Pokémon reset (HP/status) also now checks `p.megaEvolved` and, if true, looks up `POKEMON.find(pp => pp.id === p.baseId)` to restore `id`/`type`/`type2`/`ability` and clear `megaEvolved` — Mega Evolution should only last for the one battle it happened in, not the whole 3-stage run, since the player's roster (`G.playerRoster`) persists as the same object instances across stages while only HP/status get reset otherwise.
- **Mega Evolution** (added 2026-07-04, massively expanded 2026-07-05): `mega` field is optional, present on 87/153 Pokémon. `mega.spriteId` is a PokeAPI *internal* form id (not the national dex number) — sprites still resolve through the exact same `https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/{id}.png` pattern, just pointed at the mega-form id (e.g. Venusaur dex #3 → Mega Venusaur sprite id 10033). Fetch these via `https://pokeapi.co/api/v2/pokemon-species/{id}` → `varieties[].pokemon` (name contains `-mega`) → the URL's trailing id, and cross-check the mega's real type/ability from `https://pokeapi.co/api/v2/pokemon/{mega-id}` (structured JSON — far more reliable than WebFetch text-summarization for this kind of lookup, no hallucination risk). Species with two Mega forms (Charizard, Mewtwo) — this game only ships one (X form) per species; the other is simply unimplemented, not a bug.
  - **Not every internal form id in the "regular" sprite folder actually has an image, even when PokeAPI's species/ability JSON for it is complete** — verified this by curling every one of the 87 `mega.spriteId` values against `.../sprites/pokemon/{id}.png` (2026-07-05, after a user report "mega 基格爾德的圖壞了"): 86/87 resolved fine, but Mega Zygarde (`10301`) 404s on the standard path despite PokeAPI's `/pokemon/10301` endpoint returning full type/ability data — its `sprites.front_default` is literally `null`, while `sprites.other['official-artwork'].front_default` *does* exist and resolves. **Before shipping a new `mega.spriteId`, don't just trust that the species/ability data existing means the sprite does too — curl the actual PNG.** Fixed by adding a `SPRITE_ID_OVERRIDE` lookup inside `spriteUrl(id)` itself (both `pokemon_battle.html` and `public/pvp.html` — `server.js` never renders sprites, no change needed there) that redirects known-bad ids to the `other/official-artwork/{id}.png` path instead — fixing it at the single shared `spriteUrl()` function means every consumer (bench popup, victory screen, Pokédex, team-select, mad-scientist picker, the Mega cutscene reveal) gets the working image automatically, rather than needing an `onerror` fallback patched into each of the dozen-plus places that call `spriteUrl()` directly in template strings (those don't have an `onerror` handler wired at all, so a broken source there just shows the browser's broken-image icon with no recovery). `setSprite()`'s own animated-GIF→static-PNG `onerror` chain also got a third rung (falls through to `officialArtworkUrl()` too) as defense-in-depth for any future id not yet added to the override list.
  - **2026-07-05 expansion — three sources of Mega data now coexist**: (1) the original 23 species already in the 100-roster with a Gen 6/7 Mega (Venusaur, Charizard, etc. — unchanged from 2026-07-04); (2) **11 already-in-roster species** (Raichu, Starmie, Dragonite, Meganium, Feraligatr, Staraptor, Froslass, Chandelure, Golurk, Greninja, Golisopod) that gained a **first-ever** Mega via *Pokémon Legends Z-A* / its "Mega Dimension" DLC (real, released ~Oct 2025/Dec 2025-Jan 2026 — confirmed via PokeAPI, which already has this data structured under internal ids 10278-10325+, not hallucinated); (3) **53 brand-new species added to the roster specifically to give them Mega access** — 23 species that could already Mega-evolve under the *original* Gen 6/7 list but weren't in the roster yet (Pidgeot, Kangaskhan, Pinsir, Steelix, Blaziken, Sableye, Mawile, Aggron, Medicham, Manectric, Sharpedo, Camerupt, Altaria, Banette, Glalie, Salamence, Latias, Latios, Lopunny, Abomasnow, Gallade, Audino, Diancie), plus 30 species exclusive to the Z-A/Mega Dimension list (Clefable, Victreebel, Skarmory, Chimecho, Heatran, Darkrai, Emboar, Excadrill, Scolipede, Scrafty, Eelektross, Chesnaught, Delphox, Pyroar, Floette, Meowstic, Malamar, Barbaracle, Dragalge, Hawlucha, Zygarde, Crabominable, Drampa, Magearna, Zeraora, Falinks, Scovillain, Glimmora, Tatsugiri, Baxcalibur). **Lesson reinforced**: when PokeAPI or a wiki source contradicts what you "know" about which generation's Mega list is current, verify via a second independent structured source before dismissing it as hallucinated — training-data cutoff, not source error, is usually the actual explanation.
  - Several Z-A Mega abilities are **brand new, invented specifically for that game** (e.g. Mega Sol/Meganium, Dragonize/Feraligatr, Stalwart/Skarmory, Eelevate/Eelektross, Fire Mane/Pyroar) — verified their real English effect via WebSearch, but no Traditional Chinese localization exists yet anywhere crawlable (too recent), so their in-game Chinese names here are original translations, not verified official strings — flag this if the user asks specifically about official terminology. Mechanically, all of these were mapped to the closest existing ability id in this game's 21-effect library (see battle-logic skill) rather than building bespoke mechanics, consistent with this project's established "reuse over invent" pattern for abilities with no clean mechanical fit.
  - **`.mega-badge`** ("💎 可 Mega 進化") is a display-only UI element — a purple pill matching `.ability-badge`'s styling convention, rendered anywhere `poke.ability`'s badge already renders: the single-player Pokédex (`renderDex()`), `renderSelCard()` (team-select + post-stage pick screen, shared by both), the bench-switch popup (`bp-mega` element, single-player; inline template string, PvP), and PvP's team-select roster cards + mad-scientist transform-target cards. If you add a new place a Pokémon card renders, check whether it also needs this badge alongside the ability one — they're always added in the same places.
- **153 Pokémon total** as of 2026-07-05 (100 prior + 53 added in the Mega expansion above). The 100→153 batch was NOT scoped by the same final-evolution/no-legendary constraints as the earlier +30 batch — it deliberately includes legendaries/mythicals (Latias, Latios, Heatran, Darkrai, Zygarde, Magearna, Zeraora) and dragon/fairy types, because its purpose was Mega-access coverage, not roster-power balance. If asked to add more Pokémon in the future, ask which constraint set (if any) applies rather than assuming either precedent.
- **`desperate-blade` ability bug found and fixed 2026-07-05**: 5 Pokémon (Farfetch'd-strategy 蔥遊兵, 貓老大, 長毛狗, 鍬農炮蟲, plus the newly-added 摔角鷹人) all had `ability:{id:'desperate-blade', ..., desc:'HP 低於 50% 時，攻擊傷害 ×1.3'}` in their data, but the damage-multiplier chain in `doAttack()` (both engines) never actually checked for this id — the ability was purely cosmetic, doing nothing in battle, since whenever it was originally added. Fixed by adding a `halfHpSelf` check and a `desperate-blade` branch to the `abilityDmgMult` ternary chain in both `pokemon_battle.html` and `server.js`. When adding new abilities to data, grep the ability id in the damage-calc function to confirm it's actually wired up, not just present as flavor text — data and mechanic can silently drift apart otherwise.
- **Chinese name accuracy — verify, don't recall from memory.** The +30 batch (2026-07-03) shipped with ~19/30 Traditional Chinese names wrong (invented/misremembered rather than the real Taiwan localization), plus one dex-id error (assigned Golisopod id 979, which is actually some Gen-9 Pokémon — its real id is 768) and a pre-existing error in the original 70 (穿山王/Sandslash had been mislabeled 沙包蛇). All were caught by the user visually comparing sprite to name and fixed the same day by checking every entry against `https://tw.portal-pokemon.com/play/pokedex/{4-digit-id}_0` (use `_1` if `_0` 404s — some Pokémon, like Mega/Crowned forms, only exist under `_1`; also watch for internal PokeAPI form-variant ids like `10188` that aren't real dex numbers at all — cross-check with a plain web search for "{species} 全國圖鑑編號" if a fetch looks suspicious). **Before typing a Chinese Pokémon name from memory, verify it this way first** — recall accuracy for anything beyond famous Gen 1/2 species has proven unreliable.

## Type system

- 16 types: fire/water/grass/electric/psychic/fighting/ghost/dragon/steel/ice/normal/dark/ground/flying/rock/fairy, plus poison/bug (added to `EFF`/`TYPE_ZH`/`TYPE_COLOR` 2026-07-02 — they existed on some Pokémon's data already but the display maps and the `bug` attacker matchup row were missing until then; if a type ever shows as raw English text or always deals neutral damage, check whether it's missing from these maps before assuming the chart itself is wrong).
- `TYPE_COLOR` (hex per type, used for badges) and `TYPE_ZH` (Chinese display name) — simple lookup maps, same in both single-player and PvP client (`public/pvp.html` keeps its own copy for local rendering, but `EFF`/type logic on the PvP side is only used for *display* — actual effectiveness math for damage is computed server-side in `server.js`, see battle-logic skill).
- `EFF` = the raw type chart (`EFF[attackerType][defenderType] → multiplier`). `eff()` reads it; `effActive()` (single-player) / `srvEffActive()` (server.js) additionally apply stadium-card modifiers (反轉世界 inverts the whole chart, 龍之谷 caps dragon-vs-fairy/ice at ×1, 邪惡森林 changes grass's own effectiveness rules) — if you're debugging "why did this deal the wrong damage multiplier," check the active stadium before assuming the chart itself is wrong.
- Dual-type multiplies both types' effectiveness together (can reach ×4 or ×0.25).

## Abilities (特性)

**All 153/153 Pokémon have an ability as of 2026-07-05.** History: first batch of 6 shipped 2026-07-02 (HP<250 only, a one-off scoping, not a standing rule). A batch of 9 shipped 2026-07-03 alongside the +30 Pokémon addition, invented/loosely-matched to real abilities. Two further batches (19, then the remaining 45) shipped 2026-07-04 for the *original* 70 — every one of these individually verified against the Pokémon's real in-game ability via the official Taiwan site before being coded (see the name-accuracy note above — same lesson applied to abilities; many legendaries' real abilities are weather/terrain/PP-based mechanics this game has no system for, so those were deliberately reused against the closest existing effect with the real ability name kept for display — see the "no clean mechanical fit" rows below). The 53-species Mega expansion batch (2026-07-05) followed the same reuse-heavy approach for both base and Mega abilities, verified via PokeAPI's structured ability slugs rather than WebFetch summaries — no new `ability.id` mechanics were introduced, all 53 species' base and Mega abilities reuse one of the existing ~21 effect ids.

**Adding a new ability**: add the `ability:{id,name,trigger,desc}` field to the POKEMON entry (both files), then wire the actual effect — either into `triggerAttackerAbility`/`triggerDefenderAbility` (post-hit, simple), directly into `doAttack`'s multiplier chain (damage-number changes, hit interception, pre-hit-state checks), or into `tryInflictStatus`/`triggerOnEnter` (status-infliction-time or enter-time interactions like `own-tempo`/`insomnia`/`sync-status`/`pressure`) — the data alone does nothing. **Don't forget the UI won't show it either** unless the Pokémon actually reaches a render path that reads `poke.ability` (wired for all standard card/popup renders plus the single-player Pokédex tab as of 2026-07-04 — see ui-rendering skill's ability-badge section — but double check if you add a new selection screen; PvP has no Pokédex tab). Multiple Pokémon can freely share the same `ability.id` even with **different `name`/`desc` text** per entry (e.g. `blaze-boost` displays as "茂盛"/"激流"/"猛火"/"強子引擎"/etc depending on species) — the dispatch is purely by `id`, so reuse-with-relabeling is the norm whenever a real ability doesn't cleanly map to a new mechanic (very common for legendaries whose signature abilities are weather/terrain/crit/PP-based — this game has none of those systems); only invent a new `id` when no existing effect fits even loosely — but see `multiscale` above for a case where a genuine real-ability fit was worth adding new code instead of reusing something. 30 unique `ability.id` values exist as of 2026-07-05 (unchanged since 2026-07-04 night, despite 153 total Pokémon now vs. 100 then — the entire 53-species Mega expansion reused existing ids rather than inventing new ones) — this count includes the 4 Mega-exclusive mechanics (`tough-claws`, `technician`, `mold-breaker`, `trace`; see battle-logic skill's Mega Evolution section) since they can appear in `poke.ability` too, once mega-evolved. ## Move power/cost tiers (2026-07-02 rebalance)

All 70 (now 100 — the +30 batch was hand-authored directly into this same tier/cost shape rather than run through the rebalance script) Pokémon's movesets were regenerated by a one-time script (not hand-typed) that sorted each mon's original 4 moves by power, kept the bottom 2 as "weak" and top 2 as "strong", and remapped power+cost into per-tier ranges — original move names/types/status effects were preserved, only `dmg`/`cost` numbers changed:

| Tier | Weak power | Weak cost | Strong power | Strong cost |
|---|---|---|---|---|
| 1 | 30-38 | 0-1 | 65-80 | 10-14 |
| 2 | 35-42 | 1-2 | 75-95 | 13-17 |
| 3 | 40-48 | 2-3 | 85-110 | 16-20 |

Within a tier, where a specific mon lands in its range is interpolated from its *original* total move power relative to other mons in the same tier (not flat per-tier constants) — a mon that had higher power originally still lands closer to the top of its tier's new range. If you add a new Pokémon later, pick `dmg`/`cost` by eyeballing this table for its tier rather than reusing another mon's exact numbers.
